# OrbitWindow 0.1.0-alpha2

这是一个按用户需求从零编写的 rootless 越狱插件源码项目，目标设备优先按 iPhone 13 Pro / iOS 16.6.1 / NathanLR 环境设计。

## alpha2 规格

- 左下角、右下角向内斜着滑动呼出轮盘。
- 轮盘只显示在设置中选择的 App 图标。
- 选择 App 后以小窗形式打开。
- 小窗默认居中。
- 小窗尺寸默认：顶部约 2cm、底部约 4cm、左右约 1cm。
- 小窗内保持正常点击、滑动、输入。
- 禁止用户手动改变小窗大小。
- 点击小窗外部关闭小窗。
- 小窗底部显示半透明亚克力感小白条。
- 小白条上滑进入挂靠模式。
- 挂靠模式自动缩小，但应用内容仍可见。
- 挂靠小窗可自由拖动。
- 松手后自动吸附到屏幕边缘。
- 继续往边缘推，进入贴边隐藏。
- 贴边隐藏后显示小透明拉条。
- 拉条拖出恢复挂靠模式。
- 挂靠小窗单击恢复居中小窗模式。
- 点击通知后，该通知对应 App 默认以小窗模式打开，默认不限制 App。
- 在某个 App 全屏使用时发生 App 跳转，目标 App 默认以小窗模式打开，默认不限制 App。
- 如果在桌面普通打开 App，不强制小窗。
- 设置项为中文，主界面保留两个项目：选择轮盘 App、注销应用插件效果。

## 设置项

1. 选择指定的 App 以轮盘形式打开。
2. 注销应用插件效果。

“注销”指重启 SpringBoard 让插件重新加载，不是重启手机。

## 构建

把项目上传到 GitHub 仓库根目录，然后运行：

`Actions` → `Build rootless deb` → `Run workflow`

构建产物在 Artifacts 的 `OrbitWindow-rootless-deb` 中。

## 测试反馈建议

第一次测试请只选择 1 个 App，例如 Safari。然后依次测试：

1. 设置页是否能进入。
2. 是否能选择轮盘 App。
3. 左下角/右下角向内斜滑是否能呼出轮盘。
4. 点击轮盘图标是否进入居中小窗。
5. 小窗内点击、滑动、输入是否正常。
6. 键盘是否保持在小窗之外，接近全屏输入效果。
7. 点击小窗外部是否关闭。
8. 小白条上滑是否进入挂靠模式。
9. 挂靠拖动、吸边、隐藏、拉出恢复是否正常。
10. 点击通知是否小窗打开。
11. App 内跳转是否小窗打开。
12. 桌面普通打开 App 是否仍然正常全屏。

## 说明

这是 alpha 测试项目。iOS 16 的 SpringBoard/FrontBoard 私有视图结构可能因系统版本和越狱环境不同而变化，所以首次构建安装后需要实机反馈日志继续适配。
